# メルカリ仕入れウォッチャー (注文連動版)

Amazon受注管理シートの **NEW / BUY_NG** ステータスの注文を自動でメルカリ検索し、利益が出る価格の出品があればDiscordに通知するシステム。

## 動作の流れ

```
[Amazonに注文が入る] (例: クマガイソウの苗 抜き苗 3苗 / 11,480円)
        ↓
[orders シートの該当行が NEW or BUY_NG]
        ↓
[GitHub Actions が5分おきに起動]
        ↓
[商品名から検索キーワード自動抽出]
   "クマガイソウの苗 抜き苗 3苗" → ["クマガイソウ", "クマガイソウ 苗", "クマガイソウ 抜き苗", "熊谷草", ...]
        ↓
[メルカリで net_revenue (8,024円) 以下の出品を検索]
        ↓
[商品名のマッチング判定 (漢字・カナ揺らぎ対応)]
        ↓
[ヒットしたら Discord に通知]
   発送期限・利益見込み・緊急度色付き
```

## 主な機能

- **注文連動**: orders シートの status を見て自動で検索対象を決定
- **商品名自動展開**: "クマガイソウの苗 抜き苗 3苗" から「クマガイソウ」を抽出 → 苗界隈サフィックスで複数クエリ生成
- **漢字 ⇄ カタカナ揺らぎ**: 「クマガイソウ」⇄「熊谷草」を自動対応 (山野草30種以上対応)
- **揺らぎキャッシュ**: 1度生成したらシートに保存、次回以降は使い回し (新規追加時のみ生成)
- **発送期限の緊急度判定**: 期限まで2日以内 → ピンク色 + 緊急マーク
- **利益見込み表示**: net_revenue − メルカリ価格を計算して通知に表示
- **重複通知防止**: 既出品を nae_seen シートに自動記録

## セットアップ手順 (Windows)

所要時間 約25〜35分。

---

### 0. 必要なもの

- Google アカウント (orders シートが入っているもの)
- Discord アカウント (スマホアプリ)
- GitHub アカウント

---

### 1. Discord サーバーと Webhook を作成 (約5分)

#### サーバー作成

1. PC または スマホで Discord を開く
2. 左サイドバーの「+」(サーバーを追加) → 「オリジナルの作成」 → 「自分と友達のため」
3. サーバー名: 「仕入れBot」など → 作成

#### 通知用チャンネルを作成

1. 「テキストチャンネル」の「+」 → チャンネル名「mercari仕入れ」 → 作成

#### Webhook URL を発行

1. 作成したチャンネル名の右側の歯車アイコン(チャンネル編集)をクリック
2. 左メニュー「連携サービス」 → 「ウェブフックを作成」
3. 名前を「苗仕入れBot」など、アイコンも好きに設定
4. 「ウェブフックURLをコピー」 → **このURLを控えておく**

#### スマホ通知の設定

1. スマホのDiscordアプリで該当サーバーを開く
2. サーバー名長押し → 「通知設定」
3. 「**すべてのメッセージ**」を選択 (新着即届く)

---

### 2. Google サービスアカウントを作成

1. https://console.cloud.google.com にアクセス
2. 上部のプロジェクト選択 → 「新しいプロジェクト」 → 名前 `mercari-orders` → 作成
3. プロジェクト作成後、左メニュー「APIとサービス」 → 「ライブラリ」
4. 「Google Sheets API」を検索 → 「有効にする」
5. 左メニュー「APIとサービス」 → 「認証情報」
6. 上部「+ 認証情報を作成」 → 「サービスアカウント」
7. サービスアカウント名: `mercari-orders` → 「作成して続行」 → ロールはなしで「完了」
8. 一覧に表示されたサービスアカウントのメールアドレスをクリック (例: `mercari-orders@xxx.iam.gserviceaccount.com`) → **このメールアドレスを控える**
9. 「キー」タブ → 「鍵を追加」 → 「新しい鍵を作成」 → JSON → 作成 → JSONファイルがダウンロードされる
10. **このJSONは厳重に保管** (GitHub にコミット禁止)

#### スプレッドシートにアクセス権を付与

1. お使いの **amazon-orders** スプレッドシートを開く
2. 右上「共有」ボタン
3. 手順8で控えたサービスアカウントのメールアドレスを入力
4. 権限「**編集者**」 (※閲覧者ではダメ。シート追加とデータ書き込みが必要)
5. 送信

#### スプレッドシートIDを控える

URLの `https://docs.google.com/spreadsheets/d/【ここがID】/edit` の部分をコピー。

---

### 3. GitHub にリポジトリを作成

1. https://github.com/new にアクセス
2. リポジトリ名 `mercari-orders-watcher`
3. **Private** を選択
4. 「Create repository」

#### コードをアップロード

ZIPを解凍して、フォルダ内のファイル一式を:
- リポジトリ画面の「uploading an existing file」リンク → ドラッグ&ドロップ
- コミットメッセージ「初回コミット」 → Commit changes

---

### 4. GitHub Secrets を設定

リポジトリ画面 → Settings → Secrets and variables → Actions → 「New repository secret」

| Name | Value |
|---|---|
| `SPREADSHEET_ID` | 手順2で控えたスプレッドシートID |
| `GOOGLE_CREDENTIALS_JSON` | 手順2-9でダウンロードしたJSONファイルの中身全体 |
| `DISCORD_WEBHOOK_URL` | 手順1で控えたDiscord Webhook URL |

`GOOGLE_CREDENTIALS_JSON` はJSONファイルをメモ帳で開いて、`{` から `}` まで全部をコピペ。

---

### 5. 動作確認

1. リポジトリ画面 → 「Actions」タブ
2. 左の「Mercari Orders Watcher」を選択
3. 右側「Run workflow」 → 「Run workflow」(手動実行)
4. 実行をクリックしてログを確認

成功すれば:
- スプレッドシートに **`nae_cache`** と **`nae_seen`** シートが自動作成される
- 既存のNEW/BUY_NG注文に対して、メルカリで仕入れ候補を検索
- 該当があれば Discord に通知

初回は既存出品が一括で通知される可能性があるので、最初の数分はDiscordがにぎやかになるかもしれません。落ち着いたら通常運用です。

## スプレッドシート構造の前提

このシステムは `orders` シートの以下の列を読みます:

| 列 | 項目 | 用途 |
|---|---|---|
| D | ship_due_date | 発送期限 (緊急度判定) |
| **E** | **product_name** | **検索対象の商品名** |
| H | price | 販売価格 (通知表示用) |
| L | net_revenue | **仕入れ価格上限** |
| O | status | **NEW / BUY_NG のみ検索対象** |

これに合わせて以下のシートが**自動的に**作られます:

- **`nae_cache`**: 商品名 → 表記揺らぎのキャッシュ
- **`nae_seen`**: 通知済みメルカリ商品ID

---

## 表記揺らぎ対応の中身

商品名から検索キーワードを生成する際、以下のルールで「揺らぎ」を自動展開します。

### 1. ノイズ除去

```
入力: クマガイソウの苗 抜き苗 3苗
↓ "の苗" "抜き苗" "3苗" を除去
出力メイン名: クマガイソウ
```

### 2. サフィックス組み合わせ

メイン名 × 苗界隈サフィックス で検索クエリ生成:

```
クマガイソウ
クマガイソウ 苗
クマガイソウ 抜き苗
クマガイソウ 根付苗
クマガイソウ 球根
クマガイソウ 山野草
```

### 3. 漢字 ⇄ カタカナ辞書

`src/keywords.js` 内の `KANJI_KANA_MAP` に登録された対応を自動展開:

```javascript
'クマガイソウ': ['熊谷草'],
'シラネアオイ': ['白根葵'],
'カタクリ': ['片栗'],
'ホトトギス': ['杜鵑草', '杜鵑'],
// ... 30種以上
```

新しい品種が出てきて漢字対応がほしければ、ここに追記するだけで全注文に反映されます。

### 4. ひらがな展開

カタカナ品種は自動的にひらがな版も検索対象に。

### 5. マッチング判定

メルカリから取得した商品タイトルが、本当にその品種かを再判定:
- 全角英数字 → 半角
- 大文字小文字統一
- 記号・スペース除去
- 上記すべての揺らぎ表記でチェック

## 運用のヒント

### よくある運用パターン

1. **新規注文が入る** → orders シートに status="NEW" の行が追加される
2. **5分以内** に GitHub Actions が起動し、メルカリ検索開始
3. Discord に **緊急度 + 利益見込み付き** の通知が届く
4. 通知から購入してメルカリで仕入れ
5. orders シートの status を **"BUY_C"** など別のステータスに変更
6. 以降、その注文は検索対象から外れる

### 通知が来ない時のチェックリスト

- orders シートに NEW か BUY_NG の行があるか?
- net_revenue (L列) に数値が入っているか? (空・0だとスキップされる)
- Actions タブの実行ログにエラーが出ていないか?
- すでにその商品が `nae_seen` に登録されていないか? (登録済み = 通知済み)
- メルカリ側にそもそも該当出品があるか? (人気品種は数分で売れる)

### 漢字対応を増やしたい

新しい山野草・園芸種が出てきて、漢字↔カナ対応がほしくなったら:

1. `src/keywords.js` を開く
2. `KANJI_KANA_MAP` に追記:
   ```javascript
   'シャクヤク': ['芍薬'],
   'ボタン': ['牡丹'],
   ```
3. コミット → 次回実行から反映

### 緊急通知のチャンネルを分けたい

緊急(ピンク色)とそれ以外で Discord チャンネルを分けたい場合:

1. Discord で 緊急用チャンネル `#mercari緊急` を追加作成
2. それぞれに Webhook を発行
3. `src/discord.js` を改造して urgency.isUrgent によって送信先 URL を切り替え

(必要があれば改造版を作ります)

## 注意事項

- メルカリの利用規約上、スクレイピングはグレー領域です。本ツールは個人の検索作業の自動化を目的とし、過度なアクセスを避ける設計 (5分おき・クエリ間スリープ) になっています。
- メルカリのHTML構造が変わると動かなくなる可能性があります。その場合は `src/mercari.js` のセレクタ調整が必要です。
- **net_revenue ギリギリの仕入れ価格上限設定にしているため、送料込みで赤字になる可能性があります。** 心配な場合は `src/sheets.js` の `loadTargetOrders` 内で `netRevenue * 0.9` などに調整してください。
- **Discord Webhook URL は誰でも投稿可能になるため、絶対に公開しないこと。** GitHub Secrets以外には保存しない。
